# 一括実装指示書-盤上のファンタジア.md

## 優先度付きタスク一覧
1. **プロジェクト基盤構築**: Vite/Vanilla TS環境構築、PWA設定、ディレクトリ構造作成
2. **オセロコアロジック実装**: 合法手計算、石の反転、終局・勝敗判定エンジンの作成
3. **描画エンジン実装**: Canvas APIを用いた盤面、石、アニメーションの描画
4. **ゲーム状態管理とスキルシステム**: キャラクターデータ、アクティブ/パッシブスキルのトリガー機構
5. **AI対戦（Web Worker）**: 3段階の思考アルゴリズム（Lv1: ランダム, Lv2: 評価関数, Lv3: 探索）の実装
6. **UI/UX画面実装**: スタート画面、キャラ編成、ガチャ、リザルト画面の作成
7. **P2P対戦機能（PeerJS）**: シグナリングサーバー経由でのリアルタイム通信対戦の実装
8. **データ永続化とガチャ**: LocalStorageを用いた所持キャラ・凸状況の保存、ガチャ演出の実装
9. **オーディオ・最終調整**: ユーザー操作起点の音声再生、アセット圧縮、FPS最適化

## 各タスクの明確な目的
1. アドレスバー非表示のフルスクリーンPWAとして動作させる[cite: 2]
2. スキル介入に耐えうる柔軟なオセロエンジンを構築する[cite: 1]
3. 60FPSを維持しつつ、iPhone Safariで滑らかな操作感を実現する[cite: 2]
4. キャラ6体の個別スキル（アクティブ/パッシブ）と凸による強化を反映する[cite: 1, 2]
5. メインスレッドをブロックせず、iPhoneでも快適なCPU対戦を可能にする[cite: 2]
6. 縦向き専用の操作しやすいUIレイアウトを提供する[cite: 2]
7. サーバーレス環境でプレイヤー間の対戦を実現する[cite: 2]
8. ユーザーの進行状況を安全にブラウザ内へ保存し、ガチャのランダム性を担保する[cite: 1, 2]
9. iOSの音声再生制約を回避し、リッチな体験を提供する[cite: 2]

## 画面構成
*   **スタート（ホーム）画面**: 対戦開始、キャラ編成、ガチャ、ヘルプへの遷移
*   **キャラ編成・強化画面**: 6体の所持確認、凸状況の確認、リーダー選択
*   **ガチャ画面**: キャラクター排出演出、被り時の凸処理表示
*   **プレイ画面**: 8x8盤面、キャラクターアイコン、スキル発動ボタン、制限時間カウント、思考中表示
*   **リザルト画面**: 勝敗表示、獲得報酬、ホームへの戻りボタン

## 受け入れ基準（Acceptance Criteria）
*   オセロのルールが正確に適用され、置ける場所がない場合のパス処理が自動で行われること[cite: 2]
*   アクティブスキルが1対局中に各1回のみ発動でき、盤面に即座に反映されること[cite: 1]
*   凸によるパッシブスキル（時間増減、スコア補正等）が計算式に正しく組み込まれていること[cite: 1]
*   思考時間制限（15秒）が0になった際、ランダムに石が置かれるか敗北判定が行われること[cite: 2]
*   PWAとして「ホーム画面に追加」した際、独立したアプリとして起動すること[cite: 2]

## テスト手順
1. **ロジックテスト**: 全ての合法手が正しくリストアップされるか。
2. **スキルテスト**: 6体それぞれのスキルを発動し、盤面やステータスが期待通り変化するか。
3. **負荷テスト**: Web Worker上のAI思考中にUIがカクつかないか（FPS計測）。
4. **通信テスト**: 2台の端末でルームコードを入力し、同期ズレなく対局が完了するか。
5. **永続化テスト**: リロード後に所持キャラと凸レベルが保持されているか。

## タスク間の依存関係
*   タスク2（ロジック）完了後にタスク3（描画）とタスク5（AI）を開始。
*   タスク4（スキル）完了後にタスク6（UI）のプレイ画面詳細を実装。
*   タスク7（P2P）はタスク2のコアロジックが安定していることが前提。

## 最終アウトプット
*   `dist/` 内の静的ビルドファイル一式（GitHub Pages用）
*   `sw.js` (Service Worker) によるオフラインキャッシュ
*   `manifest.json` (PWA構成ファイル)

## 主要実行コマンド
*   `npm install`: 依存関係のインストール
*   `npm run dev`: ローカル開発サーバー起動
*   `npm run build`: プロダクションビルド生成
*   `npm test`: ロジックおよびユーティリティのユニットテスト実行
